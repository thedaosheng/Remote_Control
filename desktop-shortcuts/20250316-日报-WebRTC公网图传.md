## 一、项目概述

实现 Linux 端 RealSense 双目相机通过 WebRTC 推流到 MacBook 接收端，支持公网低延迟实时视频传输。

│                 │ <------> │                  │ <------> │                 │
│ RealSense 双目   │  TURN   │  39.102.113.104  │  TURN    │   浏览器/客户端  │

## 三、实现过程

### 3.1 环境准备

**安装依赖**:
```bash
# GStreamer 核心及插件
sudo apt install -y gstreamer1.0-tools gstreamer1.0-plugins-good \
    gstreamer1.0-plugins-bad gstreamer1.0-plugins-ugly \
    gstreamer1.0-libav gstreamer1.0-nice

# Python WebRTC 绑定
sudo apt install -y gir1.2-gst-plugins-bad-1.0 python3-websockets
```

### 3.2 核心问题解决

#### 问题 1: GStreamer WebRTC 绑定缺失
**症状**: `ValueError: Namespace GstWebRTC not available`
**解决**: 安装 `gir1.2-gst-plugins-bad-1.0` 包

#### 问题 2: WebSocket 连接状态检查错误
**症状**: `'ClientConnection' object has no attribute 'closed'`
**解决**: 使用 `websockets.connection.State` 检查连接状态

#### 问题 3: TURN 密码特殊字符编码
**症状**: TURN 连接失败
**解决**: URL 编码密码中的 `!` → `%21`
```python
TURN_SERVER = "turn://remote:Wawjxyz3%21@39.102.113.104:3478"
```

#### 问题 4: on-negotiation-needed 不触发
**症状**: Offer 始终不发送
**解决**: WebSocket 连接后手动延迟启动管道并触发 create-offer

### 3.3 代码结构

**发送端文件**: `/home/rhz/teleop/scripts/20250316-cc-webrtc_signaling_client.py`

**核心流程**:
1. 连接信令服务器 `ws://39.102.113.104:8765`
2. 注册为 `sender` 角色
3. 启动 GStreamer 管道采集相机
4. 触发 `create-offer` 生成 SDP
5. 通过 WebSocket 发送 Offer
6. 交换 ICE 候选
7. 收到 Answer 后完成握手

**GStreamer 管道**:
```bash
webrtcbin name=sendbin turn-server="{TURN_SERVER}"
v4l2src device=/dev/video4 do-timestamp=true
! video/x-raw,format=YUY2,width=1280,height=720,framerate=30/1
! videoconvert
! video/x-raw,format=I420
! nvh264enc preset=5 zerolatency=true bitrate=4000 rc-mode=cbr
! rtph264pay config-interval=1
! application/x-rtp,media=video,encoding-name=H264,payload=96
! queue max-size-buffers=0
! sendbin.
```
