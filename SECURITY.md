# SECURITY

> Last updated: 2026-04-26.

## ⚠️ Known exposure: LiveKit API Secret

The Aliyun LiveKit room `teleop-room` API secret pair (`teleop_key` / `teleop_secret_key_2026`) is currently **publicly exposed** in the `main` branch git history (since at least 2026-04-06). It appears in:

- `README.md` (line ~104, table)
- `HANDOFF_PROMPT.md` (2 places)
- `cloud/livekit.yaml` (active config)
- `cloud/README.md`
- `rain-orin/zedmini_livekit_sender.py` (formerly `device-a-linux/`)
- `rain-orin/generate_livekit_tokens.py`
- `rain-orin/ros2_ws/.../sender_core.py`
- `scripts/20260406-cc-zedmini_livekit_sender.py`
- `scripts/20260406-cc-generate_livekit_tokens.py`

**Risk**: anyone can sign valid JWT tokens, join the room, view video / send fake commands / consume Aliyun bandwidth.

**Mitigation in this restructure**:

- All NEW code (PR-2/3/4) reads the secret via `os.environ.get("LIVEKIT_API_SECRET")` with **no default** — fail-fast over fallback to a known-leaked value.
- `.env.example` lists required env vars without values.
- Existing committed values are NOT scrubbed in this PR (per user instruction "暂时不要动 Aliyun"). Production rotation procedure documented below.

### Rotation procedure (perform before going production)

1. Generate new secret: `openssl rand -hex 32`
2. SSH `root@39.102.113.104`, edit `/etc/livekit.yaml` `keys:` section, replace value.
3. `systemctl restart livekit.service` and verify with `journalctl -u livekit -n 50`.
4. On each client (Orin, CangJie, ASUS), update env:
   - `~/.bashrc`: `export LIVEKIT_API_SECRET=<new>` (or via `.env` file)
   - Restart any running bridge / GUI processes / systemd services.
5. Verify cross-machine reachability: ASUS GUI → Orin `cloud_to_ros_bridge` → CangJie `chassis_controller` end-to-end.
6. (Optional) Force-rotate any cached browser sessions on Vision Pro / dashboards.

## Env var policy

All bridges and GUI code MUST follow this pattern:

```python
LIVEKIT_API_KEY = os.environ.get("LIVEKIT_API_KEY")
LIVEKIT_API_SECRET = os.environ.get("LIVEKIT_API_SECRET")
if not (LIVEKIT_API_KEY and LIVEKIT_API_SECRET):
    raise RuntimeError("set LIVEKIT_API_KEY and LIVEKIT_API_SECRET env vars (see .env.example)")
```

No `os.environ.get("...", "default-value")` form for credentials.
